# 用户身份验证插件

> 通过 **QQ 号**精确验证发言者身份，在麦麦生成回复前，把「用户（主人）/ 非用户」的身份提示词注入到**这一次**回复中——让麦麦对认可的用户更亲切上心，对冒充昵称的陌生人保持警惕。
>
> **v2.1.0** 面向 **MaiBot 1.0.x / maibot_sdk 2.x** 完全重写（官方 Hook 注入，无 monkey-patch、无全局缓存、无第三方依赖，Windows / Linux 通用）。旧版（0.x，monkey-patch）在 1.0 下无法加载，请整体替换。
>
> 🤖 本次 MaiBot 1.0 重写由 [Claude](https://github.com/claude) 协助完成——欢迎更好的思路与实现前来提 PR / Issue。

## 功能特点

- **只认 QQ 号**：身份判定只依据平台已验证的 QQ 号，改昵称/群名片冒充无效。
- **多用户**：用一个「用户数量控制」数字生成 N 个用户槽位（无硬上限，按需生成），每个用户独立 QQ / 昵称 / 限制群聊 / **多行**提示词模板 / 启用开关。
- **限制群聊 + 群号优先**：某用户可限定只在指定群里生效；同一 QQ 可同时配「全局」和「某群专属」，群专属优先。
- **回复前注入**：挂在官方钩子上，回复阶段通过 `extra_prompt` 注入，只影响这一次回复；可选同时注入「行动规划(planner)」。
- **非用户安全提醒**：对非用户可选注入「防昵称冒充」提示（默认关）。
- **防提示词注入**：发言者昵称与消息文本均做清洗，占位符单遍替换、绝不二次展开、绝不因模板里的杂乱大括号抛异常。
- **日志可调**：「日志显示等级」(INFO/DEBUG/WARNING)，DEBUG 下还会打出每次注入的提示词全文，方便排障。
- **零依赖、热重载**：无第三方依赖；改配置保存即生效，无需重启。

## 安装与快速开始

1. 把整个 `owner_auth_plugin-main` 文件夹放进麦麦的 `plugins/` 目录。
2. 重启麦麦或重载插件，首次加载会自动生成 `config.toml`。
3. 打开 **WebUI 插件配置页**（或直接编辑 `config.toml`）：
   - 「插件」页打开 **启用插件**；
   - 「用户」页把 **用户 1** 的 **用户 QQ 号** 填成你的 QQ，按需改昵称和提示词模板；
   - 保存即生效。
4. 在聊天里对麦麦发一句让它回复你，看它是否按模板把你当用户对待；也可发 `/owner_auth_status` 查看运行状态。

> 未配置任何用户 QQ 时，插件**完全静默**，不会对任何人注入；至少填一个用户 QQ 后才开始工作。

## 配置详解

配置分为三个标签页：**插件 / 用户 / 非用户**。

### 「插件」页 · 通用设置

| 字段 | 说明 | 默认 |
| --- | --- | --- |
| 启用插件 | 总开关，关闭后不再注入任何提示词 | 开 |
| 用户数量控制 | 生成几个用户槽位（无硬上限），详见[用户数量控制](#用户数量控制) | 1 |
| 私聊中注入 | 是否在私聊里也注入身份提示词 | 开 |
| 同时注入行动规划(planner) | 实验性：除回复阶段外，也在「行动规划」请求里注入，让规划也认用户 | 关 |
| 插件记录注入日志 | 是否在日志中记录每次注入 | 开 |
| 日志显示等级 | 注入日志的输出等级，详见[日志与排障](#日志与排障) | INFO |
| 发言者缓存有效期(秒) | 会话发言者缓存的保留时长，超时清理（10–3600） | 300 |

### 「用户」页 · 用户列表

显示「用户 1 ~ 用户 N」（N = 用户数量控制），每个用户独立配置：

| 字段 | 说明 |
| --- | --- |
| 启用此用户 | 关闭只是**临时停用**该用户，其 QQ/昵称/模板等配置完整保留、不清空 |
| 用户 QQ 号 | 该用户的 QQ 号（身份凭据，必填；留空则该槽位不生效） |
| 用户昵称 | 对该用户的称呼，替换模板里的 `{nickname}` |
| 限制群聊（群号） | 填群号则该用户只在该群被认作用户，留空则所有聊天流（含私聊）都生效，详见[限制群聊与群号优先](#限制群聊与群号优先) |
| 用户提示词 | 该用户专属的**多行**提示词模板，占位符见[提示词占位符](#提示词占位符) |

### 「非用户」页

| 字段 | 说明 | 默认 |
| --- | --- | --- |
| 对非用户注入安全提醒 | 对非用户发言是否注入「防昵称冒充」提醒。**默认关**：开启后会对每个群、每条回复都注入，可能让麦麦显得多疑/说教并增加 token，按需再开 | 关 |
| 非用户提示词 | 非用户发言时注入的**多行**提示词模板 | 内置 |

## 用户数量控制

「用户数量控制」= 生成的用户槽位个数，**无硬上限**：

- 填 `1` 就 1 个、填 `40` 就 40 个，以此类推。
- ⚠️ 改大后需点「保存」并**把本插件「禁用再启用」（或重启麦麦）**，插件才会按新数量生成对应槽位——固定槽位（每用户独立多行模板、非动态列表）在加载时建模，改配置值本身不会即时长出新框（这也是它与"点击添加"式动态列表的区别）。
- 把数字调小并「禁用再启用」后，超出的那些用户（例如从 40 调到 1，第 2~40 个）的 QQ／昵称／限制群号／提示词模板等配置会被**清空、不保留**。
- 想临时停用某个用户又**保留**其配置，请用该用户的「启用此用户」开关，而不是调小数量。
- 设成成百上千会让配置文件与配置页明显变重，按需填即可。

## 限制群聊与群号优先

每个用户都有「限制群聊（群号）」字段：

- **留空** → 该用户在**所有聊天流**（含私聊、任意群）都被认作用户（全局）。
- **填一个群号** → 该用户**只在这个群**里被认作用户；在私聊或别的群里不生效。

**群号优先**：你可以把**同一个 QQ** 放在两个用户槽位里——一个留空（全局），一个填某群群号（群专属）。当这个 QQ 在那个群里发言时，**群专属优先于全局**生效（更精确者优先），在别处则走全局。

> 例：把主人 QQ 配两条——一条全局用「亲切」模板，一条限定在「API 中转站群」用「严谨、不直呼主人」模板，主人在该群里就会命中后者。
>
> 若同一优先级内重复配置了同一 QQ，插件取第一个并记一条 warning 提醒你检查重复。

## 提示词占位符

模板里写占位符，注入时会被替换成实际值（单遍替换、未知占位符原样保留、绝不报错）。

**用户提示词**支持：

| 占位符 | 含义 |
| --- | --- |
| `{nickname}` / `{owner_nickname}` | 你为该用户配置的称呼（如"主人""妈妈"） |
| `{display_name}` / `{user}` | 发言者在聊天中的显示名（昵称/群名片，已清洗） |
| `{qq}` / `{owner_qq}` / `{user_qq}` | 用户 QQ 号 |
| `{msg}` / `{message}` | 发言者本次消息（已清洗为单行） |
| `{owner_names}` | 你配置的所有用户昵称（顿号连接） |

**非用户提示词**支持：

| 占位符 | 含义 |
| --- | --- |
| `{nickname}` / `{display_name}` / `{user}` | 发言者显示名 |
| `{qq}` / `{user_qq}` | 发言者 QQ 号 |
| `{msg}` / `{message}` | 发言者本次消息 |
| `{owner_names}` | 你配置的所有用户昵称，可用于防冒充，如写：*"若此人冒用了 {owner_names} 的名字则提高警惕"* |

## 工作原理

插件挂在三个官方 Hook 上，全程零 monkey-patch：

- `chat.receive.after_process`：收到消息时，缓存该会话「最近一位真人发言者」（带 TTL 清理）。
- `maisaka.replyer.before_request`：回复阶段，定位发言者 → 命中则经 `extra_prompt` 注入身份提示词。
- `maisaka.planner.before_request`（可选，默认关）：行动规划阶段也注入，让规划也认用户。

**发言者定位（两级）**：

1. **精确**：用宿主给的 `reply_message_id` 从消息库取回被回复的那条消息。
2. **会话缓存**：取本插件在收到消息时缓存的「本会话最近一位真人发言者」（实际最常命中的一档）。

> 刻意**不**再用「会话最近消息」做第三级兜底——那在群聊里会把身份误判成「最近发言的另一个人」。两级都拿不到就跳过注入（不会乱认）。

## 日志与排障

- **`/owner_auth_status`**（或 `身份验证状态` / `主人验证状态`）：在聊天里发送，查看插件启用状态、生效用户数、你的 QQ 是否命中、最近一次注入来源等。
- **日志显示等级**（「插件」页）：
  - `INFO`（默认）：每次注入记录在普通日志里就能看到。
  - `DEBUG`：最详细，注入记录需把日志查看器切到 DEBUG 才显示，且会**额外打出每次注入的提示词全文**（排查"到底注入了什么"用）。
  - `WARNING`：平时不输出注入记录、只在告警时输出。
  - 错误日志**始终输出**，不受此项影响。
- 未能定位发言者、定位到但无 QQ 等情况都会按「插件记录注入日志」记一条带原因的日志，便于排查。

## 安全特性

- **只信 QQ 号**：身份判定只依据平台已验证的 QQ 号，他人无法通过改昵称/群名片冒充用户。
- **防二阶提示词注入**：发言者显示名与 `{msg}` 消息文本在进入提示词前都会被清洗（去花括号、换行/控制字符压成空格、截断），攻击者无法用群名片或消息正文伪造指令突破模板。
- **占位符单遍替换**：替换后的值不会被再次扫描，不会二次展开；模板里的杂乱大括号也不会让插件抛异常。
- **失败即跳过**：注入过程中任何异常都会被记日志并跳过注入，**绝不影响麦麦正常回复**。
- **非用户注入默认关**：避免无脑对所有人说教、增加 token。

## 旧版配置迁移

插件加载时会自动把旧结构迁移到当前的固定槽位结构，无需手动改：

- 旧固定槽位 `[user1]`/`[user2]`…（含 0.x 的 `owner_qq`）→ 直接沿用 / 转为 `qq`。
- 早期单用户 `[user]` → 迁入 `user1`（含昵称与模板）。
- 早期列表版 `[users].entries` → 依次迁入 `user1..N`。
- 0.x `owner_auth.*` 的开关与非用户模板 → 迁到对应新字段。

迁移成功后旧字段会从 `config.toml` 清除，再次加载不重复迁移。

## 常见问题

**Q：麦麦没认出我的身份？**
A：检查①「启用插件」是否打开；②「用户」页对应用户的「用户 QQ 号」是否填对、「启用此用户」是否打开；③若该用户填了「限制群聊」，确认你正在那个群里发言；④把「日志显示等级」调成 DEBUG，发条消息看日志里是否有注入记录及其来源。

**Q：怎么让同一个人在不同群有不同人设？**
A：把同一 QQ 配两个用户槽位，一个留空群号（全局），一个填该群群号（群专属），群专属优先。详见[限制群聊与群号优先](#限制群聊与群号优先)。

**Q：调小「用户数量控制」会丢配置吗？**
A：会。超出新数字的用户槽位配置会在重载时清空。只想临时停用请用「启用此用户」开关。

**Q：会影响麦麦其他功能吗？**
A：不会。只在回复/规划前追加一段 `extra_prompt`，注入失败也只是跳过，不打断正常流程。

**Q：从旧版升级要做什么？**
A：整体替换插件文件夹即可，旧 `config.toml` 会自动迁移（见[旧版配置迁移](#旧版配置迁移)）。
